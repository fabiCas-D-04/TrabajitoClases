#Principios Basicos de Rest
##1 Cliente-Servidor:
separar interfaz del usuario y su almacenamiento de datos
##2 Sin Estado:
No se guarda sesion se almacenan en cache para mejorar el rendimiento
##3 Cacheable:
Las repuestas se almaceanan en cache para mejorar el rendimiento
##4 Interfaz Uniforme:
Metodos HTTP y su estrcutura de datos 
##5 Sistema en Capas:
El cliente no necesita saber si el servidor accede directo a datos
